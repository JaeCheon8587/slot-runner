## 3. 사용자 역할
- 운영자(토스트로 개입 시점 인지 → 슬롯 PTY 직접 입력(F006)으로 답/취소 또는 종료/유지 결정)

## 4. 사전 조건
- target repo 에 Notification 훅 설치(hook_installer)
- OS 알림 권한 허용(미허용 시 콘솔 미러만)

## 5. 기본 흐름
1. claude 가 입력 대기(AskUserQuestion·권한·idle)로 멈춤 → `Notification` 훅 발화
2. 토스트 "slot-N 입력 대기/알림: …" + 콘솔 미러
3. 운영자가 토스트 보고 → 슬롯 선택 → PTY 직접 입력(F006)으로 답하거나 종료 결정 — [ADR-005](../ADR/SLOTRUNNER-ADR-005.md)
4. (별도) 슬롯이 완료/실패(outcome) 되면 토스트 "slot-N 완료/실패: 사유"

## 6. 대안 흐름
- 알림 권한 거부 → 토스트 생략, 콘솔에는 계속 기록
- 토스트 전송 실패 → 비차단(콘솔 미러로 충분)

## 7. 예외 흐름

| E1 | 알림 권한 미허용 | 토스트 생략, 콘솔만 | 없음(콘솔) | 불필요 |

## 8. 상세 기능 요구사항
- Notification 훅(kind=notification) 수신 시 토스트 + 콘솔 미러. 본문 = notification_type(+message 앞부분).
- 슬롯 outcome 이 null→done/failed 로 **전이**할 때만 토스트(중복 알림 방지).
- 토스트 제목에 출처 슬롯(panel_id) 포함.
- **자동 복구(넛지) 금지** — 멈춘 슬롯에 자동 입력 주입하지 않는다([ADR-008]).
- 권한은 1회 요청·캐시. 거부 시 이후 토스트 생략.

## 9. 입출력 개념
| 구분 | 내용 | 제약 | 예시 |
|---|---|---|---|
| 입력 | Notification 훅 이벤트 + 슬롯 outcome 전이 | 출처 슬롯(panel_id) 식별 | 없음 |
| 출력 | 데스크톱 토스트 + 콘솔 라인 | 민감정보 미표시 | `SlotRunner — slot-1 / 입력 대기` |

## 10. 상태 정의
- 알림 권한: 미요청 → 허용 | 거부(모듈 1회). 거부 시 토스트 비활성.
- 슬롯 outcome 전이 추적(이전값 대비)로 1회만 토스트.

## 11. 권한 조건
| 역할 / 권한 | 허용 작업 | 거부 시 기대 결과 |
|---|---|---|
| 운영자 | 토스트 수신, 슬롯 개입(F006) | 권한 거부 시 콘솔만 |

## 12. 데이터 처리 원칙
- 보존: 콘솔 미러(휘발). 토스트는 OS 알림 센터(앱 비보존)
- 민감정보: 토스트 본문은 유형·짧은 메시지만(로그 본문 전체 미노출)
- 부분 실패: 토스트 실패가 슬롯 동작에 영향 없음

## 13. 비기능 요구사항
