# SLOTRUNNER-FRD-003 — F007 컨텍스트 점유 기반 자동 압축(/compact)

> **기능 요구 문서**: 스텝 루프가 단계 전이 시 슬롯 세션의 컨텍스트 점유율을 측정해 임계 이상이면 `/compact` 를 자동 주입하고 압축 완료 후 다음 단계를 잇는다(F007).
> **코드 상세 금지**: 경로·파일명·클래스·설정 키 미기재(개념만).

## 3. 사용자 역할
- 운영자(콘솔로 점유율·압축 진행 감독, 임계/윈도우 설정 조정)

## 4. 사전 조건
- 스텝 루프가 슬롯에서 단계 전이를 구동(FRD-001)
- 슬롯 세션 트랜스크립트가 존재(점유 측정 원천)
- 임계 설정값(기본 40%)·컨텍스트 윈도우 설정값(기본 1,000,000 토큰) 존재

## 5. 기본 흐름
1. 슬롯이 한 단계를 끝낸다(Stop)
2. 다음 단계가 있으면, 트랜스크립트로 현재 컨텍스트 점유율(pct)을 측정한다
3. `pct < 임계` → 압축 생략, 다음 단계 지시 즉시 주입 — [ADR-007](../ADR/SLOTRUNNER-ADR-007.md)
4. `pct ≥ 임계`(또는 측정불가) → `/compact` 주입, 압축 완료 신호(SessionStart)를 기다린다
5. 압축 완료 → 대기 중이던 다음 단계 지시를 주입한다
6. 단계가 더 없으면(완료) 압축 판정 없이 종료 마감(F004)

## 6. 대안 흐름
- 측정불가(트랜스크립트 없음/파싱 실패)면 보수적으로 압축한다(과압축 허용 — 폭주가 더 위험)
- 임계가 100 이면 사실상 압축 안 함, 0 이면 사실상 매 전이 압축(설정 양 끝 의미)

## 8. 상세 기능 요구사항
- 점유 추정 = 트랜스크립트 마지막 (비 sidechain) assistant `usage` 의 input+cache 합. 분모 = 컨텍스트 윈도우 토큰(설정).
- 판정·주입은 **단계 전이 시점**(다음 단계가 있을 때만). 마지막 단계 완료에는 적용 안 함.
- 임계 미만이면 압축 생략(속도 보존). 임계 이상이면 다음 단계 **전에** `/compact`.
- 압축 완료는 SessionStart 신호로 인지, 그 후 다음 단계 주입.
- 대기 한도 초과 시 무압축 진행(자동 재시도 없음).
- 슬롯별 독립 판정·대기(격리) — [ADR-002](../ADR/SLOTRUNNER-ADR-002.md).

## 9. 입출력 개념
| 구분 | 내용 | 제약 | 예시 |

| 입력 | 슬롯 트랜스크립트(점유 원천) + 임계·윈도우 설정 + 단계 전이 이벤트(Stop) | 윈도우 > 0, 임계 0~100 | 없음 |

## 10. 상태 정의
- 슬롯(스텝 관점): 단계 실행중 → (전이) 측정 → [압축 대기] → 다음 단계 실행중. 압축 대기는 전이 직후의 임시 상태(완료 신호/한도 초과로 해소).
- 압축 대기 상태는 휘발(재시작 시 소실) — [ADR-004](../ADR/SLOTRUNNER-ADR-004.md).
