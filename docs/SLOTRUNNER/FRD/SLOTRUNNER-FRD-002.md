## 3. 사용자 역할
- 운영자(슬롯 그리드·콘솔 감독, 슬롯 선택·직접 입력으로 개입/취소, 점유 슬롯 수동 해제)

## 4. 사전 조건
- 단일 루프 구동(FRD-001)이 슬롯 1개에서 동작
- 슬롯 수 설정값 존재(기본 2), 큐 상한 설정값(기본 10)

## 5. 기본 흐름
1. 잡 수신 시 빈 슬롯 있으면 그 슬롯에 배정·구동
2. 빈 슬롯 없고 큐 < 10 이면 FIFO 큐 적재(대기 순번 응답)
3. 큐가 이미 10 이면 **거부** + 잡 명세의 Monday 링크에 **실패 댓글 작성**(QUEUE_FULL) — [ADR-002](../ADR/SLOTRUNNER-ADR-002.md)
4. 슬롯이 비면(종료) 큐에서 다음 잡 dequeue·배정
5. 각 슬롯 Hook 이벤트를 panel_id 로 식별해 공용 콘솔에 출처 태그와 함께 시간순 표시
6. 운영자가 슬롯을 클릭해 포커스 → 키 입력이 그 슬롯 PTY 로 전달(수동 개입·취소) — [ADR-005](../ADR/SLOTRUNNER-ADR-005.md)
7. 운영자가 큐 비우기를 요청하면 대기 큐 전체 제거(실행중 슬롯은 무관)

## 6. 대안 흐름
- [유지]로 슬롯이 점유된 채면 가용 슬롯이 줄어 큐 대기가 길어진다(의도) — [ADR-002](../ADR/SLOTRUNNER-ADR-002.md)
- 앱 재시작 시 진행중 슬롯·큐는 **소실**(휘발) — 봇이 재요청 — [ADR-004](../ADR/SLOTRUNNER-ADR-004.md)

## 8. 상세 기능 요구사항
- 슬롯당 정확히 1 세션. 슬롯 간 상태 공유 금지(격리).
- 큐는 FIFO. **상한 10**. 초과분은 적재하지 않고 거부 + Monday 실패 통지. 슬롯 해제 시점에만 dequeue.
- 기본 슬롯 수 = 2(설정 변경 가능, 상한은 확장 시 결정).
- 큐 비우기 = 대기 항목 전체 제거. 실행중/유지 슬롯에는 영향 없음.
- 슬롯 포커스 = 1개 슬롯을 활성 입력 대상으로 지정. 키 입력은 활성 슬롯 PTY 로만 전달.
- 공용 콘솔 = 전 슬롯 이벤트 시간순 머지 + 출처 슬롯(panel_id) 태그. **별도 필터/탭 없음**(태그만으로 구분) — Q-F005-001 결정.
- 재시작 시 슬롯·큐 휘발(영속화 없음) — [ADR-004](../ADR/SLOTRUNNER-ADR-004.md).
- 동일 잡(같은 phase) 중복 허용(중복 무시) — Q-F002-001 결정. **단 작업 공간(워크트리/브랜치)은 잡 단위로 격리**해 동시 동일 phase 충돌을 막는다(phase 단독 키 금지). 격리 키 구체는 구현 시점 결정(예: phase + 잡 식별자).

## 10. 상태 정의
- 슬롯: 비어있음 → 구동중 → 팝업(완료/실패) → 종료(해제) | 유지(점유). 활성(포커스) 여부는 직교 속성.
- 큐: FIFO, 0~10. 적재/대기/dequeue/전체비우기.
- 재시작 시 위 상태 전부 휘발(미보존).

## 12. 데이터 처리 원칙
- 보존 대상: 콘솔 이벤트 로그(세션 동안, 휘발)
- 보존 기간: 앱 수명/콘솔 비우기까지. 재시작 시 소실
- 중복 처리: 동일 잡 중복 **허용**(중복 무시). 작업 공간은 잡 단위 격리로 충돌 방지(§8)
- 부분 실패: 한 슬롯 실패가 타 슬롯에 영향 없음(격리)
- 민감정보: 로그·콘솔에 민감정보 미표시

## 14. 로그 / 알림 / 이력 정책

- 오류 기록: 슬롯 기동 실패, 큐 포화 거부
- 알림: 큐 포화·기동 실패 시 Monday 실패 댓글(NotifyPort). 완료/실패 팝업은 FRD-001
- 이력: 없음(휘발)
